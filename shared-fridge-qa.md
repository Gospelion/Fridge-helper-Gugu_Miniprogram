# 共享冰箱 MVP 验收清单

## 部署前

- 七个共享集合均设置为仅管理端可读写，并按 README 建立索引。
- `fridgeAccess`、`fridgeSync`、`contentSecurity` 和 `recognizeFood` 已重新部署。
- 使用两个不同微信账号 A、B，清理调试缓存后重新编译。

## 主流程

- A 首次进入后设置家庭昵称，旧库存和日记进入默认“我的冰箱”。
- A 创建新冰箱并生成微信邀请；B 打开分享卡片、填写昵称并加入。
- A 添加食材后，B 回到首页可以看到；B 修改数量后，A 再次进入页面可以看到新值。
- B 创建带照片的日记，A 可以预览照片；删除日记后照片不再显示。

## 离线与冲突

- 断网后添加、编辑和删除不同记录，页面立即更新并出现“待同步”提示。
- 恢复网络后待同步数量归零，另一账号可以看到全部修改，且没有重复记录。
- A、B 离线修改同一批食材，后同步的一方出现冲突；分别验证“采用云端”和“保留本地”。

## 权限与生命周期

- 普通成员不能生成邀请、改名、移除成员或删除冰箱。
- 所有者可以移除普通成员；被移除账号刷新后不再看到该冰箱。
- 有其他成员时所有者不能直接删除，必须先转让所有权。
- 转让后新所有者获得管理权限，原所有者可以退出。
- 最后一名所有者删除冰箱后，库存、日记和共享照片均被清理。
